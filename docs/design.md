# 根目录设计文档

## 设计目标

用最小维护成本提供一个清晰的仓库导航页，让使用者先在根目录完成项目识别，再进入具体子项目查看细节。

## 方案

### 1. 根目录 README 采用静态索引

根目录 `README.md` 使用一张简洁表格列出：

- 子项目目录名
- 子项目职责说明
- 子项目 README 链接

设计原因：

- 静态文档最稳定，不依赖脚本扫描目录。
- 目录级摘要比自动枚举文件更符合人工阅读需求。
- 后续新增项目时只需要补一行，维护成本低。

### 2. 根目录只保留聚合信息

根目录 README 不重复拷贝子项目的详细使用方法，而是把详细说明下放到各子项目 README。

设计原因：

- 避免根目录文档膨胀。
- 每个子项目独立维护，符合当前目录结构。
- 可以减少因为业务细节变更导致的跨文件重复更新。

### 3. README 链接依赖各子项目自维护

如果某个一级子项目缺失 `README.md`，需要先补齐，再在根目录建立跳转。

设计原因：

- 根目录索引不能指向不存在的文档。
- 把文档责任放回子项目本身，更符合独立项目边界。
