# Translate Slide 设计文档

## 设计目标

在不改变幻灯片结构的前提下，用最简单的 Apps Script 实现当前页文本互译，并把项目边界控制在单文件脚本可以清晰维护的范围内。

## 核心流程

### 1. 菜单注册

`onOpen()` 在 Google Slides UI 中注册 `Translate` 菜单，暴露中译英和英译中两个入口。

设计原因：

- 用户直接从菜单触发，无需手动运行底层函数。
- 双入口比运行时再次询问方向更直接，减少一次交互。

### 2. 当前页定位

两个菜单函数统一调用 `translateCurrentSlide_(sourceLang, targetLang)`，由该函数读取当前选中的 page，并校验它是否为 slide。

设计原因：

- 把通用校验集中到一个函数，避免重复逻辑。
- 只处理当前页，能降低误操作影响范围。

### 3. 页面元素遍历

脚本遍历当前 slide 的 page elements，只处理：

- `SHAPE`
- `TABLE`

对于表格，逐行逐列读取单元格文本；对于 shape，直接获取 text range。

设计原因：

- 这两类元素覆盖最常见的文字载体。
- 先限制范围可以保持脚本稳定，避免为了少量场景引入复杂对象分支。

### 4. 文本过滤与翻译

`translateTextRange_()` 负责：

- 跳过空文本、URL、邮箱
- 按段落拆分文本
- 根据目标语言判断是否已经是目标语言
- 调用 `LanguageApp.translate`
- 仅在确实发生变化时回写文本

设计原因：

- 先过滤再翻译，可以减少不必要调用。
- 按段落处理可以保留基本换行结构。
- 只在有变化时回写，避免无意义覆盖。

## 当前边界

- 不处理复杂嵌套对象和图片内容。
- 不做术语一致性控制。
- 不做自动测试接入设计，本次只补齐文档边界说明。
